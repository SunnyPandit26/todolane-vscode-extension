# Contributing to TodoLane

First off, thank you for considering contributing to TodoLane! It's people like you that make TodoLane such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, make one! It's generally best if you get confirmation of your bug or approval for your feature request this way before starting to code.

## Fork & create a branch

If this is something you think you can fix, then fork TodoLane and create a branch with a descriptive name.

## Get the test suite running

Make sure you're using Node.js. Install dependencies with `npm install` and run `npm run compile`.

## Implement your fix or feature

At this point, you're ready to make your changes. Feel free to ask for help!

## Make a Pull Request

At this point, you should switch back to your master branch and make sure it's up to date with TodoLane's master branch. Then update your feature branch from your local copy of master, and push it!

Finally, go to GitHub and make a Pull Request.
